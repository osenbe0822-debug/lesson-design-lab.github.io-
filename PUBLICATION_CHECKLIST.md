# 公開前チェックリスト

一般公開前に、個人情報・ローカル環境・勤務先情報が混ざっていないか確認するためのチェックリストです。

## 1. 個人を特定できる情報

- 氏名、ハンドルネーム、メールアドレス、電話番号を書いていない
- 顔写真、自宅周辺、勤務先周辺が分かる画像を使っていない
- 職歴、勤務先、学校名、地域が特定される情報を書いていない
- SNSアカウントや個人ブログへのリンクを置いていない

## 2. ローカル環境の情報

- `/Users/...` のようなPC内のパスがHTML、CSS、JavaScript、READMEに残っていない
- `tarojiosaburo` などローカルユーザー名が残っていない
- `.codex`、`generated_images` など制作環境のパスが残っていない
- 画像ファイルに不要なメタデータが残っていない

確認コマンド例:

```bash
rg -n "tarojiosaburo|/Users|Documents/HP|\\.codex|generated_images" .
```

## 3. 学校・児童・保護者の情報

- 児童名、保護者名、教職員名を書いていない
- 学校名、クラス名、具体的な校内事情を書いていない
- 成績、健康情報、相談内容、トラブルの詳細を書いていない
- 実際の指導案や内部資料をそのまま載せていない

## 4. 著作権・IB関連

- IB公式ロゴ、公式画像、公式文書の本文を無断使用していない
- 書籍や教材の文章を長く転載していない
- 学習ノートは自分の言葉で要約している
- 必要に応じて出典や参考元を記録している

## 5. 外部サービス

- 外部API、AI API、ログイン、データベースを使っていない
- フォーム入力が外部サーバーへ送信されない
- 生成AIへ貼り付ける場合の注意書きがある

## 6. 公開URL設定

- `https://example.com/` を実際の公開URLに置き換えた
- `sitemap.xml` のURLを実際の公開URLに置き換えた
- `robots.txt` のSitemap URLを実際の公開URLに置き換えた

確認コマンド例:

```bash
rg -n "example\\.com" .
```

## 7. 公開前の最終確認

- ブラウザでトップページ、教室英語、問いメーカー、セルフチェックを確認した
- お気に入り保存、チェック結果保存、削除、コピーが動く
- スマートフォン幅でも読める
- 利用上の注意・免責事項ページが見つけやすい
- 公開後も、実在の学校・児童・勤務先の情報を追記しない運用にする
